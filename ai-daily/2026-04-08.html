Google 开源 Scion,多代理编排开始从理念走向可复现实验台
github.com · GoogleCloudPlatform/scion
Scion 是个很对味的实验方向,它让多个 agent 在各自容器、各自 worktree、各自凭证里并行协作,还故意不预设太重的 orchestration 抽象,而是把协调权更多留给模型自己。这个思路很像‘先把隔离、并行、可观察性搭起来,再看什么协作模式真有效’。如果你在做 multi-agent 系统,Scion 值得盯,因为它解决的是工程现实,不是 PPT 上的 agent society。

Hippo 用生物启发式记忆给 AI Agent 补上‘该记住什么、该忘掉什么’
github.com · hippo-memory
Hippo 的思路比‘全量存档+向量检索’更像真正的记忆系统,它把 decay、retrieval strengthening、consolidation 这些脑科学味道很重的机制搬进 agent memory。项目还强调跨工具可移植,Claude Code、Codex、Cursor、OpenClaw 都能共享记忆层。这个方向很重要,因为 agent 真正的上限,越来越取决于它如何忘记噪声,而不是如何无限堆上下文。

SandMLE 试图解决 MLE Agent 训练太贵的问题,把强化学习拉回现实
arxiv.org · arXiv
做机器学习工程 agent 最大的坑,是每次验证都要跑完整数据处理、训练和评估流水线,成本比 SWE agent 高太多。SandMLE 的办法是构造大量 50 到 200 样本的 synthetic micro-dataset task,把验证时间砍掉 13 倍以上,同时在 MLE-bench-lite 和 MLE-Dojo 上比 SFT 基线更强。核心意义在于,它让 on-policy RL 终于有机会真的用于 MLE agent,而不是停留在 paper talk。
