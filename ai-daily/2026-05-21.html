S-Bus:给多 agent 共享状态加“读集追踪”,专治 stale read
arxiv.org · Hugging Face Papers / arXiv
S-Bus 针对多 agent 共享自然语言状态里的结构性竞态问题:A 读到旧状态、B 改了内容、A 再写回去,污染就开始扩散。论文用 HTTP middleware 和 DeliveryLog 自动重建每个 agent 的 read set,在提交时发现 stale-read 和写冲突。这个方向很实用,因为多 agent 失败经常不是推理不行,而是并发语义一塌糊涂。

ThoughtTrace:第一次大规模记录用户“没说出口的想法”
thoughttrace-project.github.io · ThoughtTrace / arXiv
ThoughtTrace 收集了 1058 名用户、2155 段真实多轮对话、17058 个 turn 和 10174 条自报告 thought,把“用户为什么这么问、看到回答后怎么想”变成数据。它提醒我们:只看 prompt 和 response 会漏掉大量用户目标、犹豫和反应。对个性化助手、对齐训练和产品研究来说,这比继续堆聊天日志更接近真实需求。

huggingface.co · Hugging Face Papers

Prompts are technical debt too:提示词也是技术债,而且更难管
seangoedecke.com · Sean Goedecke
Sean Goedecke 把 prompts 直接归入技术债:它们增长快、依赖上下文、行为不透明,而且小改动可能造成大幅性能波动。AGENTS.md、CLAUDE.md、skills、tool prompts 一多,系统就会进入没人能完整理解的状态。最有用的建议不是少用提示词,而是像管代码一样管它:版本、测试、评审、边界和删除机制都要有。
